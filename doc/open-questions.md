# 开放问题集 (Open Questions)

> **作用**：项目里所有 Claude 或用户都说不准的分类/归属/本体/标注问题的中央登记处。
>
> **使用规范**：
> - 每个问题有唯一 ID `OQ-NNN`（按登记顺序编号，永不复用）
> - 各 spec 文档通过 `[OQ-NNN](open-questions.md#oq-nnn)` 引用
> - 问题可有 `current_default`（暂行约定，让管线能跑）但状态保持 `open` 直到正式解决
> - 解决后状态改为 `resolved`，条目**保留**作为决策记录
> - 推迟到后续 sprint 的标 `deferred`
>
> **欢迎外部贡献**：藏传佛教 / 佛学 / 知识工程 / 文献学 领域的专家可通过 GitHub Issue 或 PR 提供解答。

## OQ-001: 鬼神 / 鬼王香夏坚 等的六道归属

**Context**：
第三品多次出现"鬼神""妖魔鬼怪"等通称，以及具名的"鬼王香夏坚"（在莲师降伏巴达霍修行院鬼王的事件中）。这些非人不属于明确的"龙"（→ 旁生）或"罗刹"（→ 阿修罗），归属六道哪一道不易判定。

**Source**：
- "莲师又降伏了无恼龙等所有龙类" → 龙 → 旁生 ✓
- "莲师示现威猛身相降伏了他（鬼王香夏坚）" → 鬼王 → ❓
- "莲师又降伏了藏地大大小小的所有凶神恶煞、妖魔鬼怪" → 通称 → ❓
- "他们雷击红山，水冲澎塘的宫殿" → 凶神恶煞 → ❓

**At Stake**：
六道归属直接影响 KG 的查询/分组（"莲师降伏的所有阿修罗"是否能查到鬼王香夏坚？"莲师降伏的所有饿鬼"呢？）。如果归类不准，下游分组就会失真。

**Current Default (v0.2)**：
- 鬼王香夏坚 → 暂归 `非人.饿鬼`
- 鬼神/妖魔鬼怪 通称 → 暂归 `非人.通称`（不强行细分六道）

**Discussion**：
- 汉传佛教里"鬼王"通常指饿鬼道之主（阎魔王/阎罗王是饿鬼道的统治者）
- 藏传"妖魔鬼怪"包含多种非人，难以一一对应到六道
- 是否应该放弃"逐一归道"的尝试，承认很多藏地非人**没有标准六道槽位**？
- 是否新增 `非人.混合` 子类容纳无法判定的通称？

**Status**: `open`

## OQ-002: 岩罗刹女（藏人始祖之一）的六道归属

**Context**：
第三品引言里，藏人起源神话："首先在湖泊逐渐减少形成郁郁葱葱的森林时期，普陀山大悲观音加持的一只猴子来到雪域，他和岩罗刹女交配，由此繁衍藏人。"

岩罗刹女（藏文 brag srin mo）是藏人始祖之一。她是非人，但具体六道归属不明：
- "罗刹"字面归阿修罗或饿鬼
- 但她又是藏地原住民、与岩石/山林相关 → 可能是地祇性质
- 她还能与猴子（本身是观音化身的旁生）生育后代 → 性质特殊
- 始祖身份本身就处于"非人 ↔ 人类"的过渡地带

**Source**：
"普陀山大悲观音加持的一只猴子……他和岩罗刹女交配，由此繁衍藏人"（§引言）

**At Stake**：
对藏人起源神话的实体建模影响整个"创世/起源"叙事的可视化。

**Current Default (v0.2)**：
- 暂归 `非人.阿修罗`（按"罗刹"传统归类）

**Discussion**：
- 民俗起源神话中的非人是否需要单独的"创世角色"子类？
- 是否承认"始祖"作为一个特殊角色（横跨非人和人类的过渡身份）？
- 类似问题以后还会出现（藏地民间信仰中的山神/水神/树神/魂魄）

**Status**: `open`

## OQ-003: 伏藏品 → 法本 的实体唯一性

**Context**：
第三品 §松赞干布 部分：「《国王遗教》隐藏成三个伏藏，由哲土哦哲、阿达尼、敦巴释迦光三位开取伏藏，当今被人们一致称为《玛尼全集》」

这里有一个本体论问题：
- 《国王遗教》（被埋藏前的状态）
- 《玛尼全集》（被掘取并整理后的状态）

它们是**同一个实体的两个状态**，还是**两个独立的实体**？

**At Stake**：
- 如果是同一实体：实体记录只有一条，但要表达"埋藏前/掘取后"的状态变化（需要时间维度的属性，而 entity 本身是不变的）
- 如果是两个实体：建立 `revealedFrom` 关系连接它们，但要面对"哪些事件归哪个实体"的问题

类似的双状态问题在伏藏传统里**极其普遍**——每一个伏藏都有"埋藏文 → 取出文"的演化。这是伏藏类目的根本性建模问题。

**Current Default (v0.2)**：
- 暂作两个实体：
  - 《国王遗教》归 `经典.密续.岩藏`
  - 《玛尼全集》归 `经典.教言集`
  - 二者通过 `revealedFrom` 关系连接

**Discussion**：
- shiji-kb 没有对应案例可参考（《史记》里没有"伏藏"概念）
- 国际佛学数据库（如 BDRC = Buddhist Digital Resource Center）的处理方式值得调研
- 第六品 近传伏藏史 进入标注后这个问题会反复出现，届时必须有定论
- 备选方案 C：单一实体 + 状态时间线（reified state changes）

**Status**: `open`

## 状态约定

| 状态 | 含义 |
|---|---|
| `open` | 未解决，暂行 default 已应用，等待解答 |
| `resolved` | 已正式决定，default 转正，条目保留作为决策记录 |
| `deferred` | 推迟到后续 sprint（数据更多时再看） |

## 如何登记新 OQ

1. 取下一个未使用的 ID（OQ-004, OQ-005, ...）
2. 在索引表添加一行
3. 在主体追加一节，包含：Context / Source / At Stake / Current Default / Discussion / Status
4. 在引发该 OQ 的 spec 文档处加 `[OQ-NNN](open-questions.md#oq-nnn)` 引用
5. commit 时 message 说明"adds OQ-NNN about ..."
