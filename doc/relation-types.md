# 关系类型规范 (Relation Types)

> **状态**：v0.1 -- **草案（待 PoC 验证）**
> **作用范围**：本项目知识图谱中所有实体间的持久关系
> **数据来源**：本规范的所有关系类型与例子均从敦珠《藏密佛教史》第三品 · 藏传佛法归纳
> **依赖**：entity-taxonomy v0.4 | narrative-layers v0.2
> **方法论**：数据驱动 -- 通读第三品全文 (~8000 字, ~191 entity)，归纳出所有重复出现的实体间连接模式，再分类命名

---

## 一、关系的定义与目的

**关系 (Relation)** 是两个实体之间的持久性结构化连接，记录为有向三元组 `(subject, predicate, object)`。

**与事件的区别**（详见第五节）：
- **关系**是持久的：松赞干布 `manifestationOf` 观世音菩萨 -- 始终成立
- **事件**是临时的：赤松德赞 `迎请` 静命 -- 发生在特定时间地点

**用途**：
- 构建人物/教法/寺院/教派之间的结构化网络
- 支撑下游的路径查询（如：从西日桑哈到贡巴绕色的传法链路）
- SKILL_09 可视化的边标签与过滤维度

**不用途**：
- 不替代事件记录（事件有时间、地点、角色等丰富上下文）
- 不替代实体属性（origin、specialty 等留在 entity record 内部）

## 三、各关系类型详细定义

### A. 化身/形态关系

#### A1. `manifestationOf`

> 已在 entity-taxonomy v0.4 R2 CASE A 定义，此处复用。

- **含义**：历史人物是某位圣众的化身
- **方向**：人物 -> 圣众
- **主体约束**：`人物.*`
- **客体约束**：`圣众.*`
- **叙述层**：通常标 `神变`（化身宣称本质上超凡）
- **第三品例子**（共 ~6 例）：
  - 赤松德赞 `manifestationOf` 文殊菩萨 -- "文殊菩萨幻化人君赤松德赞出世"
  - 松赞干布 `manifestationOf` 观世音菩萨 -- "观世音菩萨示现为人君的形象"
  - 赤热巴坚 `manifestationOf` 金刚手菩萨 | 拉托托日涅赞 `manifestationOf` 普贤菩萨 | 噶尔东赞 `manifestationOf` 金刚手 | 囤弥桑布札 `manifestationOf` 文殊菩萨

#### A2. `formOf`

> 已在 entity-taxonomy v0.4 R2 CASE B 定义，此处复用。

- **含义**：一个命名身相是另一实体的特定形态（具有独立修法/造像）
- **方向**：身相 entity -> 主 entity
- **主体约束**：`圣众.*`（通常是本尊子类）
- **客体约束**：`圣众.*` 或 `人物.*`（当主 entity 有 additional_types 时）
- **叙述层**：`神变`
- **第三品例子**：
  - 多吉卓罗 `formOf` 莲花生大士 -- "莲师示现令人望而生畏的忿怒威猛相...当时莲师被人们尊称为'多吉卓罗'"

---

### B. 师承/传法关系

#### B1. `teacherOf`

- **含义**：甲是乙的老师（传授了具体教法）
- **方向**：教师 -> 弟子
- **主体约束**：`人物.*`（法师/帝王均可）
- **客体约束**：`人物.*`
- **设计决策**：师承是**单一关系 + 类型标签**，不拆成多种独立关系。类型标签 `transmission_type` 区分传法方式。
- **必填属性**：
  - `transmission_type`：灌顶 `abhisheka` / 口传 `lung` / 戒律 `vinaya` / 教授 `teaching` / 窍诀 `upadesha`（开放集）
  - `teaching`：所传教法/经典 entity ID（可选，已知时填）
- **叙述层**：通常 `史传`（传法行为本身是凡常事件）
- **第三品例子**（共 ~15 例，选代表性 4 例）：
  - 西日桑哈 `teacherOf` 贝若札那 `{transmission_type: 教授, teaching: 大圆满}` -- "贝若札那在西日桑哈前听受大圆满"
  - 莲花生大士 `teacherOf` 君臣有缘者 `{transmission_type: 灌顶}` -- "赐予下密与无上续相应的灌顶"
  - 静命 `teacherOf` 巴绕那 `{transmission_type: 戒律}` -- 出家受戒中静命作为亲教师
  - 贡巴绕色 `teacherOf` 鲁墨 `{transmission_type: 戒律}` -- "传了一次性圆满比丘戒"

---

### C. 修建/创立关系

#### C1. `builtBy`

- **含义**：寺院/建筑由某人修建（施主、设计者或主持建造者）
- **方向**：寺院 -> 人物
- **主体约束**：`寺院.*`
- **客体约束**：`人物.*`
- **可选属性**：
  - `role`：施主 `patron` / 堪布 `khenpo` / 奠基者 `consecrator`（开放集）
- **叙述层**：通常 `史传`
- **第三品例子**（共 ~10 例，选 3 例展示 role 区分）：
  - 桑耶寺 `builtBy` 赤松德赞 `{role: 施主}` | 静命 `{role: 堪布}` | 莲花生大士 `{role: 奠基}`
  - 大昭寺 `builtBy` 松赞干布 -- "兴建起大昭寺与小昭寺"
  - 吉祥善增寺 `builtBy` 赤热巴坚 -- "在吉玛哦江多建起..."

#### C2. `foundedBy`

- **含义**：道场/教派/传承由某人创立
- **方向**：道场/教派 -> 人物
- **主体约束**：`寺院.道场` | `教派.*`
- **客体约束**：`人物.*`
- **叙述层**：`史传`
- **第三品例子**：
  - 拉木道场 `foundedBy` 鲁墨 -- "创立了拉木、绕吉、也巴..."
  - 一百零八处道场 `foundedBy` 格西札巴 -- "格西札巴独自一人也建立了一百零八处道场"

---

### D. 政治/世俗关系

#### D1. `rulerOf`

- **含义**：某人是某地的统治者
- **方向**：人物(帝王) -> 地名
- **主体约束**：`人物.帝王`
- **客体约束**：`地名.*`
- **第三品例子**：
  - 涅赤赞普 `rulerOf` 藏地 -- "藏地出现了第一位国王涅赤赞普"
  - 赤松德赞 `rulerOf` 吐蕃 -- "统治了赡部洲东方三分之二的领土"

#### D2. `fatherOf` / D5. `sonOf`

- **含义**：父子关系。`sonOf` 是 `fatherOf` 的逆关系，实际记录只选最自然的方向，另一方向由推理生成
- **方向**：父 -> 子（`fatherOf`）或 子 -> 父（`sonOf`）
- **约束**：`人物.*` -> `人物.*`
- **第三品例子**：
  - 赤松德赞 `fatherOf` [木奈赞普, 木赤赞普, 木迪赞普] | 木迪赞普 `fatherOf` 赤热巴坚
  - 涅赤赞普 `sonOf` 百军王 -- "有些认为这位国王是百军王的太子"（层：`传闻`）

#### D3. `consortOf`

- **含义**：配偶关系（对称，记录时只存一条，推理引擎补全反向）
- **约束**：`人物.*` <-> `人物.*`
- **第三品例子**：松赞干布 `consortOf` 尼泊尔赤尊公主 | 松赞干布 `consortOf` 文成公主

#### D4. `ministerOf`

- **含义**：甲是乙的大臣/辅佐
- **约束**：`人物.大臣` -> `人物.帝王`
- **第三品例子**：噶尔东赞 `ministerOf` 松赞干布 -- "法臣噶尔东赞商议"

---

### E. 译经关系

#### E1. `translatedBy`

- **含义**：经典由某位译师翻译或校勘
- **方向**：经典 -> 人物（通常 specialty 含 `译师`）
- **约束**：`经典.*` -> `人物.*`
- **可选属性**：`role` (译师 `translator` / 校勘 `reviewer`) | `partner` (合译班智达 ID)
- **第三品例子**：
  - 显密经典 `translatedBy` [贝若札那, 噶·瓦拜则, 焦·鲁坚赞, 祥·益西德]
  - 《广般若经》 `translatedBy` 嘎瓦那波切 `{role: 校勘}` -- "进行译校"

---

### F. 降伏/守护关系

#### F1. `subduedBy`

- **含义**：非人被某人降伏并立誓
- **约束**：`非人.*` -> `人物.*`
- **叙述层**：`神变` | **可选属性**：`result`（降伏后结果）
- **第三品例子**：
  - [十二地母, 十三革绕神, 二十一优婆塞] `subduedBy` 莲花生大士 `{result: 立誓守护正法}`
  - 鬼王香夏坚 `subduedBy` 莲花生大士 `{result: 保护僧财}`

#### F2. `protects`

- **含义**：护法/非人守护某寺/教法/伏藏
- **约束**：`非人.*` -> `寺院.*` | `经典.*` | `教法.*`
- **叙述层**：`神变`
- **第三品例子**：被降伏之非人 `protects` 伏藏品 -- "吩咐作为伏藏品的护法神"

---

### G. 伏藏关系

#### G1. `concealedBy`

- **含义**：伏藏经典由某人埋藏/隐藏
- **约束**：`经典.密续.岩藏.*` -> `人物.*`
- **叙述层**：`神变`（伏藏埋藏非凡常事件）
- **第三品例子**：
  - [一百国王魂伏藏, 五大意伏藏, 二十五甚深大伏藏] `concealedBy` 莲花生大士
  - 《国王遗教》 `concealedBy` 松赞干布 -- "隐藏成三个伏藏"

#### G2. `revealedBy`

- **含义**：伏藏经典由伏藏师开取
- **约束**：`经典.密续.岩藏.*` -> `人物.*`（specialty 含 `伏藏师`）
- **叙述层**：`神变` 或 `史传`
- **第三品例子**：《国王遗教》 `revealedBy` [哲土哦哲, 阿达尼, 敦巴释迦光] -- "三位开取伏藏"

---

### H. 归属关系

#### H1. `memberOf`

- **含义**：个体属于某个集合实体
- **约束**：`人物.*` -> `集合.*` | **叙述层**：`史传`
- **第三品例子**：[静命, 莲花生, 赤松德赞] `memberOf` 师君三尊 | 巴绕那 `memberOf` 预试七人

#### H2. `belongsToSect`

- **含义**：实体归属于某教派/部派
- **约束**：`人物.*` | `寺院.*` | `教法.*` -> `教派.*`
- **第三品例子**：莲花生大士 `belongsToSect` 宁玛 | 大圆满 `belongsToSect` 宁玛

#### H3. `locatedIn`

- **含义**：寺院/建筑位于某地
- **约束**：`寺院.*` -> `地名.*`
- **第三品例子**：永布拉冈 `locatedIn` 藏地 | 桑耶寺 `locatedIn` 卫藏

---

### I. 引文/典据关系

> I 类和 J 类关系连接的是**事实断言 (fact)**，不是实体。使用 `subject_fact` / `target_facts` 字段。

#### I1. `cites` -- 断言引用经典作为典据 (fact -> 经典, 层：`典据`)
- F_028 `cites` 《巴协》 -- "按照《巴协》等中'只是住了短暂的时间'"

#### I2. `refutes` -- 断言反驳另一种说法 (fact -> fact, 层：`宗派`/`史传`)
- 讷巴班智达的观点 `refutes` "从天而降"说 -- "纯属苯波传言"

#### I3. `reconciles` -- 断言调和多个矛盾异说 (fact -> fact[], 层：`宗派`)
- F_027 `reconciles` [F_025a..F_026] -- "自宗诸位智者...与这些史实都无有矛盾之处"

---

### J. 异说关系

#### J1. `alternativeOf` -- 同一主题的不同说法 (fact -> fact)
- "住了六个月" `alternativeOf` "住了十八个月" | "十二小国" `alternativeOf` "四十个零星小国"

## 四、关系记录格式 (YAML Schema)

### 4.1 实体间关系

```yaml
relations:
  - id: REL_001                     # 全局唯一 ID (REL_NNN)
    type: manifestationOf           # 本规范定义的关系类型 ID
    subject: E_P01                  # 主体 entity ID
    object: E_S01                   # 客体 entity ID
    transmission_type: null         # teacherOf 专用（必填）
    teaching: null                  # teacherOf 可选
    role: null                      # builtBy / translatedBy 可选
    result: null                    # subduedBy 可选
    layers: [神变]                   # narrative-layers 标签（必填）
    author_stance: 认可              # 认可 / 中立 / 反驳（必填）
    confidence: 0.9                 # [0.0, 1.0]（必填）
    source_quote: "文殊菩萨幻化人君赤松德赞出世"  # 必填
    source_line: 74                 # 推荐
```

### 4.2 断言间关系（cites / refutes / reconciles / alternativeOf）

```yaml
fact_relations:
  - id: FREL_001                    # 全局唯一 ID (FREL_NNN)
    type: cites
    subject_fact: F_028             # 发起断言 (fact ID)
    target: E_B_巴协                # cites -> 经典 entity ID
    target_facts: [F_025a, F_025b]  # reconciles/alternativeOf -> fact ID[]
    layers: [典据]
    author_stance: 中立
    confidence: 0.9
    source_quote: "按照《巴协》等中..."
```

> `subject`/`object` 用于实体间关系；`subject_fact`/`target`/`target_facts` 用于断言间关系。两组互斥。

## 五、与事件的关系

| 维度 | 关系 (Relation) | 事件 (Event) |
|------|----------------|--------------|
| 时间性 | 持久 / 无时间边界 | 有起止时间 |
| 结构 | 三元组 (S, P, O) | 多槽位（角色/地点/时间等）|
| 适用场景 | "X 是 Y 的弟子" | "X 于某年在某地向 Y 求法" |

**关系可从事件中抽取**（如：求法事件 -> `teacherOf` 关系），**但反方向不成立**（关系无法还原事件的时间地点上下文）。

## 六、已知局限

| # | 局限 | 说明 |
|---|------|------|
| 1 | **第三品范围限制** | 所有关系类型仅从第三品归纳。后续章节可能需要新增类型（如辩论关系、供养关系等）。届时升版本号 |
| 2 | **transmission_type 取值待稳定** | 师承关系的 `transmission_type` 目前是开放集。PoC 验证后根据实际频次决定是否改为封闭集 |
| 3 | **对称关系只记录一次** | `consortOf` 是对称的，记录时只存一条。推理引擎需要自动补全反向 |
| 4 | **逆关系不重复存储** | `fatherOf` 与 `sonOf` 是逆关系对。实际记录时只选最自然的方向，另一方向由推理生成 |
| 5 | **断言间关系依赖 fact ID** | `cites` / `refutes` / `reconciles` / `alternativeOf` 需要先有 fact 记录才能建立。关系标注在事实标注之后进行 |
| 6 | **通称非人无法建立关系** | 依据 entity-taxonomy 原则 #16，"凶神恶煞"等无名非人不建 entity，因此无法作为关系的 subject/object。这些交互记录在事件中 |
| 7 | **跨章关系待定** | 同一实体在不同章节的关系如何合并（如松赞干布在第二品和第三品的关系），需要跨章 merge 策略 |
