# 实体分类规范 (Entity Taxonomy)

> **状态**：v0.1 草案 — **待用户审核**
> **作用范围**：本项目从源典抽取的所有实体（用于 SKILL_03a 实体标注）
> **数据来源**：本规范的所有类目与例子均取自敦珠《藏密佛教史》第三品 · 藏传佛法
> **方法论**：**数据驱动**——Claude 没有事先设定类目数。流程是：通读第三品 → 抽取每一个具名实体 → 自然聚类 → 命名类目 → 用真实例子验证边界 → 标注模糊/边界案例

---

## 一、用途与读者

本规范定义本项目知识图谱里"什么是一个实体""有哪些种类""如何区分类目"。

**用途**：
- SKILL_03a 实体标注的输入规范
- SKILL_05 关系构建时的实体类型约束
- SKILL_06 本体构造的种子分类
- 应用层（SKILL_09）的可视化分类色板

**不用途**：
- 不是完整本体（属性、约束、推理规则在 SKILL_06 阶段构造）
- 不是 RDF/OWL 形式化表达（v0.x 阶段保持 markdown）

## 二、方法论：如何得到这 12 个类目

```
通读第三品全文 (8000 字)
   ↓
抽取所有具名/可命名实体 (~180 个)
   ↓
按"在文本里以同样方式被引用"做自然聚类
   ↓
命名每个聚类，得 12 个候选类目
   ↓
针对每个类目，找 5+ 真实例子验证
   ↓
对边界模糊的实体，记录为"已知模糊"，由用户裁决
   ↓
v0.1 草案
```

**关键约束**：类目数不预设。Claude 没有强行凑成 18 类或 23 类，最终是 12 类——这是数据自然聚类的结果。

## 三、核心原则

| # | 原则 |
|---|---|
| 1 | **数据驱动**——任何类目必须有第三品里的至少 3 个真实例子；只有 1-2 例的暂归"其他"，等数据多再独立 |
| 2 | **类目与 narrative-layers 正交**——一个实体的类目（如"人物.法师"）与关于该实体的事实的叙述层（如"神变层"）是两个独立维度 |
| 3 | **化身关系不重叠类目**——"X 是观音的化身"不会让 X 同时归"人物"和"圣众"。X 还是 X 的本来类目（人物.帝王），化身关系单独建模为 `manifestationOf` |
| 4 | **别名归一**——一个实体可以有多个表面形式（菩提萨埵 = 静命 = 萨霍堪布菩提萨埵绕杰达），它们指向同一个实体节点 |
| 5 | **类目可演化**——后续章节可能新增/拆分/合并类目，需要时升版本，不要回避 |
| 6 | **角色不是类目**——亲教师/羯磨师/译师 等是角色（role），由关系类型表达，不是实体类目 |
| 7 | **位次不是类目**——持明位次/六种任运持明/信解行菩萨阶位 等是属性（attribute），不是实体类目 |
| 8 | **集合是实体**——师君三尊、预试七人、祖孙三法王 等有文化身份的群体是独立实体（集合类目），不只是引用 |

## 四、12 个类目总览

| # | 类目 | 第三品例子数 | 主要子类 | 一句话定义 |
|---|---|---|---|---|
| 1 | **人物** | ~80 | 帝王/王妃/法师/译师/伏藏师/空行母/施主/其他 | 历史上的具体个人 |
| 2 | **圣众** | ~7 | 佛/菩萨/本尊/护法 | 本尊体系中的圣者（修行/皈依对象） |
| 3 | **非人** | ~15 | 鬼神/龙/罗刹/天/具名非人 | 佛教宇宙论里的非人类生灵（通常被降伏） |
| 4 | **寺院** | ~16 | 寺庙/王宫/道场/修行处 | 建筑类宗教/政治场所 |
| 5 | **地名** | ~50 | 国/区域/圣地/神话地理/自然地理 | 地理位置 |
| 6 | **经典** | ~22 | 显宗经/密宗续/论释/史籍/教言集 | 文本类知识载体 |
| 7 | **教法** | ~30 | 教法体系/修法分类/具体教言 | 知识内容（不是文本，是文本承载的法） |
| 8 | **教派** | ~5 | 藏传宗派/印度部派/外道/子派 | 教派/学派/组织实体 |
| 9 | **仪轨** | ~10 | 灌顶/修法/供养/占卜 | 行为模板（reusable ritual procedures） |
| 10 | **法器/圣物** | ~10 | 佛像/法器/圣物 | 物质性的宗教物品 |
| 11 | **伏藏** | ~4（指代为主） | 单一伏藏/伏藏集合/伏藏法本 | 莲师/前师埋藏待掘的法宝 |
| 12 | **集合** | ~10 | 上师集合/弟子集合/王族集合 | 有文化身份的群体引用 |

**类目数 = 12**——介于 shiji-kb 的 23 类（更多因为有官职/谥号等汉地特有类）和最简的 7-8 类之间。

## 五、各类目详细

### 1. 人物 (Person)

**定义**：具有自身姓名的历史上的具体个人。一个实体节点 = 一个人。

**子类**（粒度细一些，因为人物是最大类）：

| 子类 | 说明 | 第三品代表例子 |
|---|---|---|
| 1.1 帝王 | 赞普/法王/国王 | 涅赤赞普、松赞干布、赤松德赞、赤热巴坚、朗达玛 |
| 1.2 王妃/王族 | 王妃、太子、公主 | 尼泊尔赤尊公主、文成公主、木奈赞普、木赤赞普 |
| 1.3 大臣/施主 | 大臣、国舅、施主 | 噶尔东赞、国舅札玛 |
| 1.4 法师 | 班智达/堪布/上师/比丘 | 静命（菩提萨埵）、莲花生大士、布玛莫札、贡巴绕色 |
| 1.5 译师 | Lotsawa/翻译者 | 囤弥桑布札、贝若札那、噶·瓦拜则、焦·鲁坚赞 |
| 1.6 伏藏师 | Tertön/掘藏者 | 哲土哦哲、阿达尼、敦巴释迦光 |
| 1.7 空行母 | Dakini（作为历史人物的空行母，不是本尊） | 益西措嘉 |
| 1.8 其他 | 婆罗门、外道、汉地和尚、传说人物等 | 婆罗门祥嘎绕、和尚玛哈雅那、和尚格旺、和尚吉番 |

**识别特征**：
- 有具体姓名（人名/法名/尊号）
- 在叙事中作为行动主体出现
- 通常有出生地/活动地/师承等可追溯属性

**强制属性**：
- `name` ：主名（最常用的称呼）
- `aliases` ：别名集合
- `sub_type` ：上述 8 子类之一

**建议属性（可选）**：
- `origin` ：印度/藏地/汉地/尼泊尔/邬金/其他
- `dates` ：生卒年（多为不详，可空）
- `sect` ：所属教派
- `roles` ：在事件中担当的角色集合

**与其他类目的边界**：
- 莲师 是 1.4 法师，**不是** 圣众。即使他被宁玛派视为"第二佛"，他的实体节点是历史人物。"莲师 是 阿弥陀佛/观音 的化身"用 `manifestationOf` 关系表达
- 益西措嘉 是 1.7 空行母（历史人物子类），**不是** 圣众。某些密法的"空行母"作为本尊是另一回事

---

### 2. 圣众 (Holy Beings / Yidam)

**定义**：佛教本体论中的圣者——佛、菩萨、本尊、护法。它们不是历史上有出生死亡的具体个人，而是修行/皈依/观想的对象。

**子类**：

| 子类 | 说明 | 第三品代表例子 |
|---|---|---|
| 2.1 佛 | 佛陀（本师/过去/未来佛、报身佛） | 释迦牟尼（本师真佛）、迦叶佛 |
| 2.2 菩萨 | 大乘菩萨 | 观世音菩萨（大悲观音、自现圣者观音）、文殊菩萨、普贤菩萨、金刚手菩萨 |
| 2.3 本尊 | 密法修持的本尊（事/行/瑜伽/无上瑜伽部） | 颦眉度母、度母 |
| 2.4 护法 | 护持佛法的本尊型护法 | 多闻天子（毗沙门） |

**识别特征**：
- 名字带"佛""菩萨""本尊""护法"等后缀
- 在叙事中以"加持""化身""幻化""示现"等方式与历史人物互动
- 不会被作为"出生于某年""卒于某地"的描述对象

**强制属性**：
- `name`、`aliases`、`sub_type`

**建议属性**：
- `manifestation_of`：是某更高层次本尊的化身（如"文殊菩萨"是诸佛智慧的化身）
- `family`：属于哪个本尊家族（莲花部、金刚部等）

**与其他类目的边界**：
- 莲花生 不属于圣众——他是 1.4 法师（有历史活动）
- 多闻天子 既是本尊又是护法，归 2.4 护法（其作为本尊的属性单独记）
- 以"X 菩萨/佛"在文本中现身（不是化身）的对象 → 圣众

### 3. 非人 (Non-human Beings)

**定义**：佛教/藏传宇宙论中的非人类生灵——鬼神、龙、罗刹、天神、空行（非历史人物的）等。区别于"圣众"，非人通常**不是修行对象**，而是被降伏/调伏/守护的对象。

**子类**：

| 子类 | 说明 | 第三品代表例子 |
|---|---|---|
| 3.1 通称 | 类的概念（非具名个体） | 鬼神、龙类、罗刹、天神、妖魔鬼怪、非人 |
| 3.2 具名非人 | 有名字的具体非人 | 无恼龙、鬼王香夏坚、罗刹王绕卡土创、十二地母、十三革绕神、二十一优婆塞 |
| 3.3 神话起源角色 | 创世/起源叙事中的非人 | 岩罗刹女（藏人始祖之一）、那只猴子（观音化身但作为类） |

**识别特征**：
- 文本中以"鬼神""龙""罗刹""天神""非人"等称呼
- 行动通常涉及被降伏、立誓守护、协助/危害人类
- 和"圣众"的区别：本尊是被礼敬/观想的，非人是被降伏的（虽然降伏后可能成为护法）

**强制属性**：
- `name`、`sub_type`

**与其他类目的边界**：
- "降伏后立誓守护"的非人在身份上变化——但仍归非人类目，"成为护法"是关系（`becameProtector`）而非类目变更
- 真正的本尊型护法（多闻天子等）归 圣众.护法，不归这里
- 神话/历史的边界：藏人始祖"猴子"和"岩罗刹女"是神话人物，归非人.3.3，不归人物

---

### 4. 寺院 (Monastery / Built Site)

**定义**：建造的、有空间边界的、宗教或政治用途的建筑实体。

**子类**：

| 子类 | 说明 | 第三品代表例子 |
|---|---|---|
| 4.1 寺庙 | 正式的佛教寺院/经堂 | 大昭寺、小昭寺、镇肢寺、镇节寺、桑耶寺（吉祥桑耶无变自成寺）、嘎穹多吉扬寺、吉祥善增寺、邬金敏珠朗佛学院 |
| 4.2 王宫/宫殿 | 帝王居所 | 永布拉冈、玛热王宫、澎塘宫殿 |
| 4.3 道场/讲学院 | 讲学/修行的非寺院场所 | 拉木、绕吉、也巴、瓦耶、凑迪、巴达霍修行院、因相乘讲学院、密宗修行院 |
| 4.4 修行处 | 自然或半自然的修行点 | 玛龙多吉札绕安穹那炯、丹迪寺（兼属 4.1）、门卡内让狮子堡、达仓三十处 |

**识别特征**：
- 名字带"寺""院""宫""堡""堂""塔"等后缀
- 文本中作为"建造""开光""保存""毁灭""驻锡"的对象出现
- 通常关联到一位或多位上师/帝王

**强制属性**：
- `name`、`aliases`、`sub_type`、`location`（地名引用）

**与其他类目的边界**：
- 寺院 vs 地名：寺院是**建筑**，地名是**地理位置**。一个寺院"位于"一个地名（建寺关系）
- 神话宫殿（莲花宫殿、铜色吉祥山顶的宫殿）→ 归寺院 4.2 但加 `mythic=true` 属性
- 邬丹达波热经堂（印度，作为桑耶寺范本）→ 寺庙

### 5. 地名 (Place / Geography)

**定义**：地理位置的引用——国、区域、圣地、自然地理、神话地理。

**子类**：

| 子类 | 说明 | 第三品代表例子 |
|---|---|---|
| 5.1 国/政体 | 国家、王朝、政体 | 印度、汉地、吐蕃、尼泊尔、邬金、龟兹国、香巴拉、革拉夏、葛逻、霍尔境、萨霍 |
| 5.2 大区域 | 跨多个具体地点的地理区域 | 雪域/藏地、卫藏、后藏、阿里、多康、康区、南赡部洲/赡部洲 |
| 5.3 圣地/具体地 | 有名字的具体地点（修行地、行政地、地标） | 普陀山、雪域如波神山口、赞塘阁希、红山、芒耶贡塘、札马翁布园、黑波山、青普、贡塘拉土、吉曲尼占帕翁卡、洞巴戒创、舍卫城 |
| 5.4 神话地理 | 佛教宇宙论中的地理 | 须弥山、四大部洲、八小洲、铁围山、罗刹洲、妙拂洲、铜色吉祥山、罗刹八洲 |
| 5.5 自然地理 | 山、湖、河、岩、园 | 札马湖、札达瓦岩山、宿喀（无溪水之地）、阿秀沙漠、卓多三境 |

**识别特征**：
- 文本中以"在……地方""到了……""从……来""位于……"等结构出现
- 名字带"山""湖""沙漠""国""境""洲""地"等
- 不像"寺院"那样是建造的实体

**强制属性**：
- `name`、`aliases`、`sub_type`

**建议属性**：
- `parent_region`：所属上级地理区域
- `coordinates`：可选，如有现代对应

**与其他类目的边界**：
- 圣地 vs 寺院：圣地是地理点，寺院是建筑。一个圣地可以有多个寺院
- "三隐蔽王处""五山谷""二十一修行地""二十座雪山" 等集合地名 → 5.3 子类，作为单一实体，并标注 `cardinality`
- 神话地理与现实地理混在同一句时分别打标

### 6. 经典 (Text / Canon)

**定义**：成文的知识载体——经、续、论、注疏、教言集、史籍、伏藏文等。

**子类**：

| 子类 | 说明 | 第三品代表例子 |
|---|---|---|
| 6.1 显宗经 | 显宗经典（佛说） | 《百拜忏悔经》、《宝箧经》、《六字真言》、《广般若经》、《般若十万颂》、《般若》、《贤愚经》、《毗奈耶经》、《无垢天女授记经》 |
| 6.2 密宗续 | 密宗续部 | 《文殊根本续》、《大圆满心部》、《普作王续》、《金刚橛》法部 |
| 6.3 论/释 | 论藏、注疏 | 《窍诀见鬘论》、《语言学书籍》八本 |
| 6.4 史籍 | 史书/年代史 | 《巴协》 |
| 6.5 教言/伏藏文 | 教言集、伏藏开取的文 | 《国王遗教》、《玛尼全集》、《吉祥证道歌》 |
| 6.6 经典集合 | 典籍集合的引用（不是单部书） | 三藏、续部、上分对法、下分对法、显密经典 |

**识别特征**：
- 用《》标记
- 文本中以"翻译""讲""听受""传讲""引用""撰著"等动词出现
- 通常关联到译师或祖师

**强制属性**：
- `name`、`aliases`、`sub_type`

**建议属性**：
- `language_origin`：原语（梵/藏/汉）
- `translator`：译师（如有）
- `category_doctrine`：所属教法体系

**与其他类目的边界**：
- 经典 vs 教法：经典是**承载教法的文本**（实体），教法是**文本所讲的法**（内容）。例：《大圆满心部》是经典，"大圆满"是教法
- 经典集合（"三藏""续部"）作为单独子类 6.6，不与单部经典混淆
- 伏藏开取出来的法本（如《国王遗教》→《玛尼全集》）归经典 6.5；伏藏作为待开取的"埋藏物"归类目 11

### 7. 教法 (Teaching / Doctrine)

**定义**：知识内容——教法体系、修法分类、具体教言、戒律范畴等。区别于经典（文本载体），教法是文本承载的内容。

**子类**：

| 子类 | 说明 | 第三品代表例子 |
|---|---|---|
| 7.1 教法体系 | 大类教法 | 大圆满、真实意法、八大法行、大悲寂猛教言、观音法 |
| 7.2 修法分类 | 修法的阶段/层次 | 生起次第、圆满次第、下密、无上续 |
| 7.3 教法范畴 | 显/密、大/小、前译/后译 | 显宗、密宗、前译密宗、大乘、小乘、续部 |
| 7.4 戒律/律仪 | 戒律范畴 | 八关斋戒、十善（法）、十八界、戒律传承、东律传承 |
| 7.5 教言/法门 | 具体教言集合 | 在家道德规范十六条、二规法令 |
| 7.6 转法轮分类 | 三转法轮的子类 | 中转法轮般若类、末转法轮 |

**识别特征**：
- 不带《》（《》是经典）
- 文本中以"修""学""听受""讲""灌顶""传授""得"等动词出现
- 名字通常是描述性短语（"大圆满""八关斋戒"）而非书名

**强制属性**：
- `name`、`aliases`、`sub_type`

**建议属性**：
- `vehicle`：所属乘（声闻/独觉/菩萨/金刚）
- `tantra_class`：续部分类（事/行/瑜伽/无上瑜伽）

**与其他类目的边界**：
- 教法 vs 经典：见 §6
- 教法 vs 仪轨：教法是知识内容，仪轨是行为模板。"大圆满"是教法，"金刚舞"是仪轨
- 教法 vs 教派：教法是知识，教派是组织。"大圆满"是教法，"宁玛派"是教派（虽然宁玛派以大圆满为主）

---

### 8. 教派 (Sect / School)

**定义**：教派/学派/僧团/法系等组织实体。这些是"人能属于的"宗教组织，区别于教法（人能学习/传授的知识）。

**子类**：

| 子类 | 说明 | 第三品代表例子 |
|---|---|---|
| 8.1 藏传宗派 | 藏传佛教主要教派 | 苯教（苯波教）、噶当派；以及（虽未在第三品出现但属本类目）宁玛、噶举、萨迦、格鲁、觉囊 |
| 8.2 印度部派 | 印度小乘部派 | 说一切有部 |
| 8.3 印度外道 | 非佛教派别 | 外道六本师 |
| 8.4 子派/法系 | 教派内的子传承 | 东律传承（戒律子传承）、前译宁玛 |

**识别特征**：
- 名字带"派""教""宗""部""乘""传承"等
- 文本中以"属于……派""依……传承""创立……派""毁灭……教"等形式出现

**强制属性**：
- `name`、`aliases`、`sub_type`

**建议属性**：
- `founder`：创派祖师
- `founding_date`：创立年代

**与其他类目的边界**：
- 与 narrative-layers 中的 `宗派(派别)` 是**同一概念**——这里的实体节点 = 那里的 `sect` 参数值
- 教派 vs 集合：教派是组织实体（持续的），集合是历史群体引用（一次性的，如"师君三尊"）

### 9. 仪轨 (Ritual)

**定义**：可重复执行的宗教行为模板——灌顶、修法、供养、占卜、加持等。

**子类**：

| 子类 | 说明 | 第三品代表例子 |
|---|---|---|
| 9.1 灌顶 | 密法灌顶 | 灌顶（generic）、《金刚橛》法部灌顶 |
| 9.2 修法仪轨 | 禅修/瑜伽修持 | 金刚舞、烧施、火施、命咒 |
| 9.3 供养仪轨 | 供养仪式 | 四大供养 |
| 9.4 占卜/感应 | 感应类仪轨 | 圆光（通过镜面/水面看见非人） |
| 9.5 戒律仪轨 | 出家/受戒仪轨 | 出家戒、比丘戒、近圆戒、顿圆传戒 |
| 9.6 法事 | 寺院法事 | 开光 |

**识别特征**：
- 文本中以"修""做""举行""传授""受"等动词出现
- 通常是动作性描述（"做了两次""举行奠基仪式"）
- 不像教法那样是抽象内容，是具体的"程序"

**强制属性**：
- `name`、`sub_type`

**与其他类目的边界**：
- 仪轨 vs 教法：仪轨是程序模板，教法是知识内容。一个教法体系（如大圆满）下有多个仪轨
- 仪轨 vs 事件：仪轨是模板（reusable），事件是具体的一次执行（如"赤松德赞 21 岁迎请莲师举行的桑耶寺奠基仪式"）。事件由 SKILL_04 抽取，仪轨在这里登记

### 10. 法器 / 圣物 (Sacred Object)

**定义**：物质性的宗教物品——佛像、法器、经函、圣物。

**子类**：

| 子类 | 说明 | 第三品代表例子 |
|---|---|---|
| 10.1 佛像/圣像 | 佛/菩萨/本尊像 | 自现圣者观音、八岁佛像、十二岁佛像 |
| 10.2 法器 | 修法用的器具 | 禅杖（莲师所用）、（第三品具体法器较少；金刚杵/嘎巴拉等留待后续章节） |
| 10.3 圣物 | 其他宗教物品 | 宝箧、金塔、如意珠总持印模、经函（generic）、法衣、石碑 |

**识别特征**：
- 物质性的宗教物品
- 文本中以"迎请""供养""装藏""留下""打开"等动词出现
- 通常有可见形态

**强制属性**：
- `name`、`sub_type`

**与其他类目的边界**：
- 法器/圣物 vs 经典：经函是物质性的"经书"，归 10.3；其内容是经典 §6
- 法器/圣物 vs 伏藏：伏藏是埋藏中的圣物，但带有"待开取"的特殊属性，单独为类目 §11

### 11. 伏藏 (Terma)

**定义**：莲师/前师埋藏、待未来伏藏师开取的法宝。区别于法器/圣物，伏藏带有**时间维度的特殊语义**——埋藏时间、授记时间、开取时间、取者、有缘法主等。

**子类**：

| 子类 | 说明 | 第三品代表例子 |
|---|---|---|
| 11.1 单一伏藏品 | 具体一个伏藏 | （第三品多用集合引用） |
| 11.2 伏藏集合 | 群体引用 | 一百国王魂伏藏、五大意伏藏、二十五甚深大伏藏 |
| 11.3 伏藏法本 | 伏藏开取后的法本 | 《国王遗教》（→ 后变为《玛尼全集》） |

**识别特征**：
- 文本中带"伏藏""隐藏""开取""授记"等关键词
- 通常关联到莲师（埋藏者）+ 伏藏师（取者）

**强制属性**：
- `name`、`sub_type`、`hidden_by`（埋藏者，必填）

**建议属性**：
- `hidden_at`：埋藏地（地名引用）
- `taken_by`：开取者（伏藏师引用）
- `prophecy`：授记内容
- `revealed_form`：开取后变成的法本（经典引用）

**与其他类目的边界**：
- 伏藏 vs 法器/圣物：伏藏的"待开取 + 授记 + 取者"特殊语义使它必须独立类目
- 伏藏开取出来的"法本"（如《玛尼全集》）归经典 6.5，原本的"伏藏品"实体仍归这里
- 第三品的伏藏案例较少（主要是引用集合），主战场是第六品 近传伏藏史

### 12. 集合 (Collective)

**定义**：有文化身份的群体引用——指代一组特定的人，但作为一个整体被反复引用，本身具有独立的"身份"。

**子类**：

| 子类 | 说明 | 第三品代表例子 |
|---|---|---|
| 12.1 上师集合 | 一组以集合身份被引用的上师 | 师君三尊（静命+莲师+赤松德赞）、祖孙三法王（松赞干布+赤松德赞+赤热巴坚） |
| 12.2 弟子集合 | 弟子群体 | 预试七人、五比丘、四顶梁之柱弟子、八大栋梁弟子、三十二椽木弟子、千数短木弟子 |
| 12.3 王族集合 | 王族群体 | 海中莲花两位王妃（赤尊+文成）、天赤七王、上当二王、地上勒六王、地德八王、赞王五代 |
| 12.4 其他集合 | 其他文化身份的群体 | 三大律师（玛尔/约/藏）、祥氏三师 |

**识别特征**：
- 名字带数字 + 名词（"三尊""七人""五比丘""八大""三十二""千数"）
- 文本中以集合身份被反复引用
- 成员通常可以列举出来

**强制属性**：
- `name`、`sub_type`、`members`（成员列表，可空）、`cardinality`（成员数）

**与其他类目的边界**：
- 集合 vs 单个人物：集合是 reified（具象化）的群体引用，每个成员仍是独立的人物实体
- 集合 vs 教派：集合是历史群体（一次性的，时间局部的），教派是持续的组织
- "祖孙三法王" 是集合，因为它有文化身份；但"赤松德赞的三个儿子"是关系（描述）不是集合

## 六、命名变体（alias）处理

第三品里**一个实体多个名字**的现象极其普遍。下面是真实案例：

| 主名 | 别名/变体 | 说明 |
|---|---|---|
| 静命 | 菩提萨埵、堪布菩提萨埵绕杰达、印度瑜伽师、亲教师（在巴赤则出家事件中） | 同一印度上师的多个称呼 |
| 莲花生大士 | 莲师、多吉卓罗（忿怒相称号）、莲花生 | 同一上师的多个称呼 |
| 涅赤赞普 | 肩座王 | 同一国王的两个称号 |
| 赤热巴坚 | 赤德祖丹 | 同一国王的两个称号 |
| 朗达玛 | 秃头王 | 同一国王的两个称号 |
| 木迪赞普 | 色纳勒劲云 | 同一国王的两个称号 |
| 巴赤则 | 花扬、巴绕那 | 出家前后法名变化 |
| 莫色拉巴 | 释迦·格巴绕色、贡巴绕色大师 | 出家前 → 法名 → 后世尊称 |
| 囤弥桑布札 | （文殊菩萨化身） | 化身宣称是关系，不是别名 |
| 吉祥桑耶无变自成寺 | 桑耶寺、拉萨桑耶寺 | 同一寺院的繁简称呼 |

**处理原则**：
1. 每个实体节点有 1 个 `name` 主名 + N 个 `aliases`
2. 主名选择：选择**最常见、最简洁**的称呼（多为今日通用的简称）
3. 别名包括：异译、尊号、法名、化身名、繁简称
4. **化身关系不归别名**——"X 是 Y 的化身"是 `manifestationOf` 关系，不是 X 的别名
5. **同名异人**问题在 SKILL_03b 消歧阶段处理（第三品里没有明显冲突）

## 七、与 narrative-layers 的协同

实体类目和叙述层是**正交**的两个维度。一个事实断言同时拥有：
- **实体类目**：断言涉及的实体属于哪类（如"莲花生" → 人物.法师）
- **叙述层标签**：该断言的证据类型（如 神变 + 授记）

**示例**：

```yaml
# 断言：莲花生大士示现忿怒相降伏鬼神
fact: "莲花生大士示现忿怒相降伏藏地凶神恶煞"
entities:
  - name: 莲花生大士
    type: 人物.法师
  - name: 藏地凶神恶煞
    type: 非人.通称
layers: [神变]            # 因为示现+降伏是超凡事件
author_stance: 认可
confidence: 0.90
```

```yaml
# 断言：囤弥桑布札创藏文
fact: "囤弥桑布札被派到印度学习声明和文字，撰著藏文字形和《语言学书籍》八本"
entities:
  - name: 囤弥桑布札
    type: 人物.译师
  - name: 印度
    type: 地名.国
  - name: 《语言学书籍》八本
    type: 经典.论释
layers: [史传]            # 派遣学习+撰著是凡常事件
author_stance: 认可
confidence: 0.90
```

**注意**："文殊菩萨的化身囤弥桑布札"这一**化身宣称**是另一条单独的断言：
```yaml
fact: "囤弥桑布札是文殊菩萨的化身"
entities:
  - name: 囤弥桑布札
    type: 人物.译师
  - name: 文殊菩萨
    type: 圣众.菩萨
relation: manifestationOf
layers: [神变]            # 化身宣称归神变
```
