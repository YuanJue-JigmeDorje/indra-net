# 实体分类规范 (Entity Taxonomy)

> **状态**：v0.3 草案 — **待用户审核（规则层面）**
> **作用范围**：本项目从源典抽取的所有实体（用于 SKILL_03a 实体标注）
> **数据来源**：本规范的所有类目与例子均取自敦珠《藏密佛教史》第三品 · 藏传佛法
> **方法论**：数据驱动 + 规则化的迭代——v0.1 聚类得出 12 类目；v0.2 整合佛教传统分类（11 类目）；v0.3 引入 R1/R2/R3 三条可被自动化执行的规则，把"个案判断"转为"规则统一应用"
> **审核重心**：v0.3 起，用户审核 spec 的规则与原则；不再逐实体确认。规则错误时回退/迭代，但不接受"逐个 entity 决定"

---

## 一、用途与读者

本规范定义本项目知识图谱里"什么是一个实体""有哪些种类""如何区分类目"。

**用途**：
- SKILL_03a 实体标注的输入规范
- SKILL_05 关系构建时的实体类型约束
- SKILL_06 本体构造的种子分类
- 应用层（SKILL_09）的可视化分类色板

**不用途**：
- 不是完整本体（属性、约束、推理规则在 SKILL_06 阶段构造）
- 不是 RDF/OWL 形式化表达（v0.x 阶段保持 markdown）

## 二、方法论：数据驱动 + 用户审核迭代

```
v0.1 流程：
通读第三品全文 → 抽取所有具名实体 → 自然聚类 → 命名 12 类目 → 标注 15 个边界模糊 → 用户审核

v0.2 流程：
用户对 v0.1 给出 4 处结构性修改 + 多个判决 →
   - 圣众改用佛教传统三圣众结构
   - 非人改用六道（除人道）+ 护法 子类
   - 伏藏并入经典作为子类
   - 法师子类合并，多身份用 specialty 属性
→ 重写相应类目 → 已解决项归档，模糊项登记到 doc/open-questions.md → v0.2
```

**关键约束**：类目数不预设。从 v0.1 的 12 类调整到 v0.2 的 11 类，是数据驱动 + 传统分类双重校准的结果，不是为了凑数。

## 三、核心原则

| # | 原则 |
|---|---|
| 1 | **数据驱动**——任何类目必须有第三品里的至少 3 个真实例子；只有 1-2 例的暂归"其他"，等数据多再独立 |
| 2 | **类目与 narrative-layers 正交**——一个实体的类目（如"人物.法师"）与关于该实体的事实的叙述层（如"神变层"）是两个独立维度 |
| 3 | **化身关系不自动重叠类目**——"X 是观音的化身"本身不会让 X 同时归"人物"和"圣众"。化身关系单独建模为 `manifestationOf`。但 X 仍可能因 R1 而被多类目共存（见原则 #11） |
| 4 | **别名归一**——一个实体可以有多个表面形式，它们指向同一个实体节点 |
| 5 | **别名严格定义**——别名只包含 true alternative names（异译/法名/尊号/简繁称），**不包含** generic 角色称谓（如"印度瑜伽师""亲教师"），后者由 specialty 属性或关系上下文捕捉 |
| 6 | **类目可演化**——后续章节可能新增/拆分/合并类目，需要时升版本，不要回避 |
| 7 | **角色不是类目**——亲教师/羯磨师/译师 等是角色（role），由关系类型表达，不是实体类目 |
| 8 | **位次不是类目**——持明位次/六种任运持明/信解行菩萨阶位 等是属性（attribute），不是实体类目 |
| 9 | **集合是实体**——师君三尊、预试七人、祖孙三法王 等有文化身份的群体是独立实体（集合类目），不只是引用 |
| 10 | **不强行解决模糊**——分类不准的实体登记到 `doc/open-questions.md` 作为 `OQ-NNN`，spec 处用 `[OQ-NNN](open-questions.md)` 引用，不假装已决 |
| 11 | **实体可多类目共存**（v0.3 引入）——默认单类目；满足 R1（见 §三 bis）时可多类目共存。例：益西措嘉 = 人物.空行母 + 圣众.大乘.空行 |
| 12 | **不从外部知识填充字段**（v0.3 引入）——文本未支持的字段留空。例外：原则 #11 经 R1 显式允许的传统知识。其他所有"我猜"都不允许 |
| 13 | **集合实体支持成员升级**（v0.3 引入）——cardinality > 1 的集合 entity，未来某成员被具名提及时可升级为独立 entity，集合 entity 仍保留作为引用 |
| 14 | **跨语言名称暂不入 entity record**（v0.3 引入）——梵/藏/中三语对应等待专门的"名义对应"映射表，当前 entity record 只用源典语言（汉译本）的名字 |
| 15 | **文本忠实度 > 外部知识完整性**（v0.3.1 引入，最重要）——藏传佛教浩瀚无边，**追求 100% 准确不现实**。默认从源文本提取，承认 KG 不完整。外部知识（教派传统、修法体系、跨语言对应）不进自动管线。若需外部知识，由用户手动 override 并标记 `_source: manual`。这条原则覆盖前面所有原则的冲突 |

## 三 bis、自动化判定规则（R1 / R2 / R3）

v0.3 引入的三条规则，目的是把"个案判断"转为"标注 Agent 可以一致执行的规则"。SKILL_03a 实体标注阶段直接 invoke 这些规则。

### R1 — additional_types 何时附加（**v0.3.1 简化**）

```
默认：实体只有一个 primary type。

ADD additional_types 当且仅当：
  源文本明确把该实体描述为多个类目的对象（双模式叙述）。
  例：文本既以历史人物方式描述（出生、行动、圆寂等凡常事件）
     又以圣众方式描述（"六种任运持明""于一切刹土中同时示现"等
     必须诉诸密法本体论的属性）→ 多类目共存

判定流程（标注 Agent 执行）：
  扫描源文本中关于该 entity 的所有提及。
  - 全部是凡常人事件 → 单类目（人物类）
  - 全部是超凡描述 → 单类目（圣众类）
  - 既有凡常又有超凡描述 → 多类目（人物 + 圣众）
```

**v0.3.1 废弃的旧标准**：
> v0.3 的 R1(b) 条款"该实体在源典所属教派传统中有已建立的修法"已**废弃**。
> 原因：Claude 无法可靠判断这种细节知识（例：蒋扬钦哲旺波全集中有
> 贝若札那的上师修法，但这种细节超出 LLM 的可靠召回范围）。
> 追求 100% 准确不现实——藏传佛教浩瀚无边。

**manual override 机制**：
```
用户可基于外部知识为特定实体添加 additional_types：
  additional_types: [圣众.大乘.X]
  additional_types_source: manual
  override_reason: "蒋扬钦哲旺波全集 vol N 含此本尊修法"
```
manual override 是 KG 的"专家增强层"，与自动管线分离。

**反例**：松赞干布是观音化身，但文本只描述他作为帝王的凡常事件
（即位、迎娶、建寺、闭关）——化身宣称是 manifestationOf 关系，不是
对他作为圣众的描述。所以 NOT 多类目。

### R2 — 化身形态何时成为独立 entity

```
GIVEN 文本提及 "X 是 Y 的某种身相/形态/化身"：

CASE A — manifestationOf 关系（默认）：
  情形：X (历史人物) 是 Y (本尊) 的化身——单向化身
  → X 和 Y 是各自独立 entity
  → 关系：X manifestationOf Y
  例：松赞干布 manifestationOf 观世音菩萨

CASE B — 独立 form entity（仅当满足条件时）：
  情形：源文本引入新名字描述 Y 的某种身相，且该身相有
        独立的修法 / 造像 / 咒语（即独立的 practice identity）
  → 为该身相创建独立 entity
  → type: 圣众.大乘.本尊 (或其他 圣众 sub-type)
  → 关系：form_of: <主 entity>
  例：多吉卓罗 (圣众.大乘.本尊) form_of: 莲花生大士

判定准则：
  该 form 是否有独立 sadhana/iconography？
    是 → CASE B (独立 entity)
    否 → 仅作为主 entity 的 alias 或上下文描述
```

**适用范围**：
- 莲师八变：每变都有独立 sadhana → CASE B（每变独立 entity）
- 文殊一切智 / 妙吉祥：根据传统是否独立 → CASE B 或 alias
- 普通的"X 化身为 Y" 一次性叙述 → CASE A

**第三品执行结果**：第三品仅出现"多吉卓罗"一变，按 CASE B 创建独立 entity。其他七变不预先填充（按原则 #12）。

### R3 — 顶级 vehicle 的 vehicle 字段

```
GIVEN 一个教法 entity，若它本身就是九乘之一：

vehicle 字段留空（不自我引用）。

判定（自动执行）：
  实体的 name 或 aliases 中是否包含九乘之一：
    [声闻乘, 缘觉乘, 菩萨乘, 
     事续, 行续, 瑜伽续,
     玛哈瑜伽, 阿努瑜伽, 阿底瑜伽]
  是 → 该实体是顶级 vehicle，vehicle 字段留空
  否 → 填上其所属九乘之一
```

**第三品执行结果**：
- 大圆满（aliases 含"阿底瑜伽"）→ vehicle 留空
- 八大法行 → vehicle: 玛哈瑜伽
- 真实意法 → vehicle: 玛哈瑜伽
- 大圆满心部（属于阿底瑜伽，但不是阿底瑜伽本身）→ vehicle: 阿底瑜伽

## 五、各类目详细

### 1. 人物 (Person)

**定义**：具有自身姓名的历史上的具体个人。一个实体节点 = 一个人。

**子类**（v0.2 简化：法师不再细分译师/伏藏师等子类，改为 specialty 属性）：

| 子类 | 说明 | 第三品代表例子 |
|---|---|---|
| 1.1 帝王 | 赞普/法王/国王 | 涅赤赞普、松赞干布、赤松德赞、赤热巴坚、朗达玛 |
| 1.2 王妃/王族 | 王妃、太子、公主 | 尼泊尔赤尊公主、文成公主、木奈赞普、木赤赞普 |
| 1.3 大臣/施主 | 大臣、国舅、施主 | 噶尔东赞、国舅札玛 |
| 1.4 法师 | 一切教界出家人/在家修行者；专长用 specialty 属性区分 | 静命、莲花生、贝若札那、贡巴绕色、囤弥桑布札 |
| 1.5 空行母 | 历史上的空行母（dakini，作为人物存在的） | 益西措嘉 |
| 1.6 其他 | 婆罗门、汉地和尚、传说人物等 | 婆罗门祥嘎绕、和尚玛哈雅那 |

**法师 (1.4) 的 specialty 属性**（多选标签，不互斥）：

```
specialty 取值（开放集，可扩展）:
├── 班智达 (pandita - 学者)
├── 堪布 (khenpo - 寺院教席)
├── 上师 (lama / guru)
├── 比丘 (bhikshu - 受具足戒)
├── 沙弥 (novice)
├── 译师 (lotsawa)
├── 伏藏师 (tertön)
├── 持藏 (sutra holder)
├── 金刚持 (vajradhara - 密法上师)
├── 瑜伽士 (yogin)
├── 阿阇黎 (acharya)
├── 大成就者 (mahasiddha)
└── ...
```

**示例**：
```yaml
- name: 贝若札那
  type: 人物.法师
  specialty: [译师, 大圆满成就者]
  
- name: 静命
  type: 人物.法师
  aliases: [菩提萨埵, 萨霍堪布菩提萨埵绕杰达]
  specialty: [班智达, 堪布]
  origin: 萨霍 (印度)
  # 注：第三品里"印度瑜伽师""亲教师"是角色称谓，不是别名（见原则 #5）
```

**识别特征**：
- 有具体姓名（人名/法名/尊号）
- 在叙事中作为行动主体出现
- 通常有出生地/活动地/师承等可追溯属性

**强制属性**：
- `name` ：主名（最常用的称呼）
- `aliases` ：别名集合（严格定义，见原则 #5）
- `sub_type` ：上述 6 子类之一

**建议属性**：
- `specialty` ：对法师子类，多选标签
- `origin` ：印度/藏地/汉地/尼泊尔/邬金/其他
- `dates` ：生卒年（多为不详，可空）
- `sect` ：所属教派

**与其他类目的边界**：
- 莲师 是 1.4 法师，**不是** 圣众。"莲师 是 阿弥陀佛/观音 的化身" 用 `manifestationOf` 关系表达
- 益西措嘉 是 1.5 空行母（历史人物），**不是** 圣众.大乘.空行（后者是出世间空行护法）

### 2. 圣众 (Holy Beings)

**定义**：佛教三乘圣者的总称——声闻圣众、缘觉圣众、大乘圣众。圣众不是历史上有出生死亡的具体个人（虽然某些圣众有传记），而是修行/皈依/观想/礼敬的对象。

**v0.2 重写说明**：v0.1 用了简单的 4 子类（佛/菩萨/本尊/护法）。v0.2 按佛教传统的"三乘圣众"重新组织。

**子类结构**：

```
圣众
├── 2.1 声闻圣众（声闻四果）
│   ├── 初果 (须陀洹 srotāpanna)
│   ├── 二果 (斯陀含 sakṛdāgāmin)
│   ├── 三果 (阿那含 anāgāmin)
│   └── 四果 (阿罗汉 arhat)            ← 注意：只有第四果才是阿罗汉
├── 2.2 缘觉圣众
│   ├── 缘觉
│   └── 辟支佛
└── 2.3 大乘圣众
    ├── 2.3.1 佛               ← 释迦牟尼、迦叶佛、过去七佛、阿弥陀、弥勒等
    ├── 2.3.2 菩萨             ← 观音、文殊、普贤、金刚手、地藏等
    ├── 2.3.3 本尊             ← 度母（含颦眉度母）、胜乐金刚、时轮金刚等
    ├── 2.3.4 空行             ← 出世间智慧空行（≠ 历史空行母）
    └── 2.3.5 护法             ← 出世间护法 / 智慧护法 (≠ 非人.护法 即世间护法)
```

**第三品代表例子**：

| 实体 | 子类 | 备注 |
|---|---|---|
| 释迦牟尼 / 本师真佛 | 2.3.1 佛 | "迦叶佛教法时成为养鸡女三子"中的本师 |
| 迦叶佛 | 2.3.1 佛 | 过去佛 |
| 观世音菩萨 / 大悲观音 | 2.3.2 菩萨 | 第三品出现频率最高的圣众 |
| 文殊菩萨 | 2.3.2 菩萨 | 化身赤松德赞、囤弥桑布札 |
| 普贤菩萨 | 2.3.2 菩萨 | 化身拉托托日涅赞 |
| 金刚手菩萨 | 2.3.2 菩萨 | 化身赤热巴坚、噶尔东赞 |
| 度母（含颦眉度母） | 2.3.3 本尊 | 颦眉度母作为度母的形态/别名 |
| 多闻天子 | 2.3.5 护法 | 出世间护法（智慧护法） |

**护法的二分**（重要）：
- **出世间护法 / 智慧护法 = 圣众.大乘.护法**——已证悟的护法本尊（如多闻天子、玛哈嘎拉等）
- **世间护法 = 非人.护法**——受降伏后立誓守护佛法的非人（如十二地母、十三革绕神等），见 §3

**空行的二分**（同理）：
- **圣众.大乘.空行** = 出世间智慧空行
- **人物.空行母** = 历史空行母（如益西措嘉）
- 部分"食肉空行"等属于 非人，不是圣众

**强制属性**：
- `name`、`aliases`、`sub_type`

**建议属性**：
- `manifestation_of`：是某更高层次本尊的化身
- `family`：所属本尊家族（莲花/金刚/佛/宝/羯磨 五部等）

**与其他类目的边界**：
- 莲花生 不属于圣众——他是 1.4 法师（有历史活动）
- 益西措嘉 不属于圣众——她是 1.5 空行母
- 多闻天子 是圣众.大乘.护法（出世间），不是 非人.护法（世间）

### 3. 非人 (Non-human Beings)

**定义**：佛教/藏传宇宙论中的非人类生灵。区别于"圣众"，非人通常是被降伏/调伏/守护的对象。**注意**：旁生道（动物）也属于非人。

**v0.2 重写说明**：v0.1 用通称（鬼神/龙/罗刹/天）作子类。v0.2 按"六道（除人道）+ 护法 + 通称"重新组织。

**子类结构**：

```
非人
├── 3.1 天部 (天道众生 / deva)
├── 3.2 阿修罗 (asura，含罗刹/夜叉等食人非人)
├── 3.3 旁生 (animals - 龙/兽/鸟/虫/有意义的动物)

### 6. 经典 (Text / Canon) — 含伏藏

**定义**：成文的知识载体——经、律、论、密续、伏藏文、净相法、史籍、教言集等。

**v0.2 重写说明**：v0.1 把伏藏作为独立类目 §11。v0.2 把伏藏并入经典，作为密续的子分类（远传 vs 岩藏）。这符合佛教传统的"教法源流"分类——伏藏本质是密续的传承方式之一。

**子类结构**：

```

├── 6.2 律 (vinaya / 律藏)        ← 戒律典

├── 6.4 密续 (tantra)
│   ├── 6.4.1 远传密续 (bka' ma)  ← 师徒口传相承
│   └── 6.4.2 岩藏 (gter ma) = 伏藏
│       ├── 6.4.2.1 地伏藏 (sa gter)
│       ├── 6.4.2.2 意伏藏 (dgongs gter)
│       └── 6.4.2.3 虚空伏藏 (mkha' gter)
├── 6.5 净相法 (dag snang chos)   ← 清净观见所得
├── 6.6 史籍                      ← 史书、年代记
├── 6.7 教言/口诀集                ← 教言、口诀汇编
└── 6.8 经典集合 (引用)            ← 三藏、续部 等集合引用

### 7. 教法 (Teaching / Doctrine)

**定义**：知识内容——教法体系、教言、戒律、修法元素等。区别于经典（文本载体），教法是文本承载的内容。

**v0.2 重写说明**：v0.1 用了 6 个子类。v0.2 简化为 4 个子类，并引入**正交分类属性**处理"九乘""三转法轮"等交叉分类。

**子类**：

| 子类 | 说明 | 第三品代表例子 |
|---|---|---|
| 7.1 教法体系 | 大类教法，可作为修学主线 | 大圆满、真实意法、八大法行、大悲寂猛教言、观音法 |
| 7.2 教言/法门 | 具体教言/法门集 | 在家道德规范十六条、二规法令 |
| 7.3 戒律体系 | 戒律范畴/律仪 | 八关斋戒、十善、东律传承 |
| 7.4 修法元素 | 修法的层次/元素/资格 | 生起次第、圆满次第、灌顶（generic） |

**正交分类属性**（同一教法可同时打多个，处理交叉分类问题）：

```yaml
教法的正交属性 (orthogonal classification):
- vehicle (九乘):     声闻 / 缘觉 / 菩萨 / 事续 / 行续 / 瑜伽续 / 玛哈瑜伽 / 阿努瑜伽 / 阿底瑜伽
- turn (三转法轮):    初转 / 二转 / 三转
- section:            显 / 密 / 前译 / 后译
- sect_affiliation:   宁玛 / 噶举 / 萨迦 / 格鲁 / 觉囊 / ...
```

**示例**：
```yaml
- name: 大圆满
  type: 教法.教法体系
  vehicle: 阿底瑜伽
  section: 密
  sect_affiliation: [宁玛]
  
- name: 八大法行
  type: 教法.教法体系
  description: |
    宁玛巴所修的"出世五法行 + 世间三法行"——
    出世五法行：文殊身（毗卢部）、莲花语（弥陀部）、真实意（不动部）、
              甘露功德（宝生部）、橛事业（不空成就部）
    世间三法行：召遣非人、猛咒诅詈、供赞世神
    是宁玛巴修行者修习生起次第八种法行所依的主要本尊。
  vehicle: 玛哈瑜伽
  section: 密
  sect_affiliation: [宁玛]
```

**关于"中转法轮般若类""上分对法""下分对法"**：

这些**不是单一教法实体**，而是**分类性表达**：
- "中转法轮般若类" = 所有 turn=二转 的般若类教法的集合
- "上分对法 / 下分对法" = 阿毗达摩的两种部分

它们应该**通过 turn / category 属性表达**，不需要独立 entity。v0.1 把它们当作教法实体的做法在 v0.2 修正。

**强制属性**：
- `name`、`aliases`、`sub_type`

**建议属性（正交分类）**：
- `vehicle`、`turn`、`section`、`sect_affiliation`

**与其他类目的边界**：
- 教法 vs 经典：见 §6
- 教法 vs 仪轨：教法是知识内容，仪轨是行为模板。"大圆满"是教法，"金刚橛灌顶"是仪轨
- 教法 vs 教派：教法是知识，教派是组织。"大圆满"是教法，"宁玛派"是教派

### 处理原则

1. 每个实体节点有 1 个 `name` 主名 + N 个 `aliases`
2. 主名选择：选择**最常见、最简洁**的称呼
3. 别名包括：异译、尊号、法名、文化命名、出家前后名、同实体不同形态
4. **化身关系不归别名**——`manifestationOf` 关系
5. **角色称谓不归别名**——specialty 属性 或 关系上下文
6. **同名异人**问题在 SKILL_03b 消歧阶段处理
