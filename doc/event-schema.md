# 事件 Schema 规范 (Event Schema)

> **状态**：v0.1 — **草案**
> **作用范围**：本项目从源典抽取的所有事件（用于事件识别与结构化）
> **数据来源**：本规范的所有事件类型与例子均归纳自敦珠《藏密佛教史》第三品 · 藏传佛法
> **依赖**：`doc/entity-taxonomy.md` v0.4（实体分类）、`doc/narrative-layers.md` v0.2（叙述层标签）

---

## 一、事件的定义与目的

### 什么是一个"事件"

一个事件是一个**有行为主体、核心行为、可定位于时空的叙事单元**。它包含：
- 一个或多个行为主体（entity 引用，不限于人物——原则 #19）
- 一个核心行为或状态变化
- 可选的时间、地点、因果

### 目的

事件是连接实体与叙述层的**桥梁**。实体描述"谁/什么"，叙述层描述"在什么认知框架下"，事件描述"发生了什么"。三者交汇构成知识图谱的叙事骨架。

### 粒度控制

| 太粗 | 适中 | 太细 |
|------|------|------|
| "松赞干布弘法" | "松赞干布建大昭寺" | "大昭寺奠基" |
| "莲师入藏" | "莲师降伏十二地母" | "莲师束缚某一鬼神" |
| "朗达玛灭法" | "拉隆降伏朗达玛" | "拉隆射箭" |

**判断标准**：能用 4-8 字概括、能写 2-4 句描述、不超过 8 句 → 粒度合适。

## 二、事件类型清单

以下 14 个类型从第三品实际文本归纳而来。每个事件标注**唯一**主类型。

### 总览

| # | 类型 | 代码 | 说明 |
|---|------|------|------|
| 1 | 迎请 | `invitation` | 迎请上师、佛像、法物入境 |
| 2 | 出家受戒 | `ordination` | 出家、受戒、传戒 |
| 3 | 建寺 | `construction` | 兴建寺院、宫殿、佛塔、道场 |
| 4 | 灌顶传法 | `transmission` | 灌顶、传法、宣讲教言、听受 |
| 5 | 译经 | `translation` | 翻译经典、校勘、创制文字 |
| 6 | 降伏非人 | `subjugation` | 降伏鬼神、龙类、立誓守护 |
| 7 | 授记 | `prophecy` | 佛/本尊/祖师发出预言 |
| 8 | 埋藏伏藏 | `terma_concealment` | 隐藏伏藏品、指定护法 |
| 9 | 即位执政 | `enthronement` | 国王即位、执掌国政 |
| 10 | 离藏 | `departure` | 上师/班智达离开藏地 |
| 11 | 弘法利生 | `dharma_activity` | 综合性弘法活动、弘扬教法、修行成就 |
| 12 | 毁灭佛教 | `dharma_destruction` | 灭法、破坏寺院、驱逐僧人 |
| 13 | 恢复佛教 | `dharma_restoration` | 恢复衰退的佛法、重建僧团 |
| 14 | 婚姻联姻 | `marriage` | 迎娶王妃、联姻 |

### 各类型详细与第三品真实例子

**1. 迎请 `invitation`**
- 「迎请萨霍的堪布菩提萨埵绕杰达」——赤松德赞迎请静命
- 「从印度迎请阿阇黎革萨绕和婆罗门祥嘎绕」——松赞干布迎请印度学者
- 「由自己化现的比丘阿嘎玛德西日迎请自现圣者观音」——松赞干布迎请佛像
- 「又从印度请来嘉伟西年、塔坚钦毕拉等……诸位金刚持上师」——赤松德赞请来班智达

**2. 出家受戒 `ordination`**
- 「巴赤则出家为僧并受了比丘戒，取名为花扬」——预试七人之首出家
- 「巴色朗、巴够贝若札那……出家，受了比丘戒，以上七位就是众所共称的预试七人」
- 「贡巴绕色上师在一坐垫上以三启白次第传了一次性圆满比丘戒」——后弘期传戒
- 「接着有国舅札玛及有聪明才智的民众大约三百人出家」

**3. 建寺 `construction`**
- 「以须弥山、四大部洲……的模式建起的吉祥桑耶无变自成寺宇」——建桑耶寺
- 「推翻罗刹女仰面的要地，建起镇肢寺与镇节寺……兴建起大昭寺与小昭寺」
- 「赤热巴坚……共建造了一千所寺院」
- 「在吉玛哦江多建起了无与伦比的吉祥善增寺」

**4. 灌顶传法 `transmission`**
- 「莲花生大士也为君臣中有缘分的人们赐予下密与无上续相应的灌顶」
- 「亲教师静命为个别人传授八关斋戒，并宣讲了十善、十八界相应的法门」
- 「贝若札那在西日桑哈前听受大圆满」
- 「国王自身也是在暗地里为众多有缘者传授大悲寂猛教言」

**5. 译经 `translation`**
- 「翻译了大量的三藏及续部，创立起了佛法的轨范」——松赞干布时期译经
- 「所有译师们将显密经典、最主要的注疏论著通通翻译成藏语」——赤松德赞时期
- 「囤弥桑布札被派到印度学习声明和文字，后来以梵文作为蓝本，撰著了藏文字形」
- 「对先代时翻译的大多数佛典重新以白话文进行校勘、抉择」——赤热巴坚时期校译

**6. 降伏非人 `subjugation`**
- 「莲师降伏了十二地母、十三革绕神、二十一优婆塞为主的一切大力非人」
- 「依靠禅定把所有鬼神紧紧束缚住，让他们立誓守护正法」
- 「莲师又降伏了无恼龙等所有龙类，使他们协助国王」
- 「莲师示现令人望而生畏的忿怒威猛相，降伏了藏地大大小小的所有凶神恶煞」

**7. 授记 `prophecy`**
- 「《无垢天女授记经》中授记说："我入灭后两千五百年佛法将弘扬于红面人之境内。"」
- 「当时他得到"再过五个朝代以后会有人对经中含义心领神会"的授记」
- 「松赞干布曾亲自授记"从现在起到第五代时……出现正法。"」
- 「莲师……对国王赐予了许多未来的授记」

**8. 埋藏伏藏 `terma_concealment`**
- 「莲师又隐藏了一百国王魂伏藏、五大意伏藏、二十五甚深大伏藏为主的不可胜数的伏藏品」
- 「所有言教综合一起的《国王遗教》隐藏成三个伏藏」
- 「莲师吩咐作为伏藏品的护法神」——指定伏藏护法

**9. 即位执政 `enthronement`**
- 「人们设立木座，把他扛在肩上迎请回去，由此被共称为涅赤赞普」
- 「他在十三岁时执掌国政」——松赞干布即位
- 「国王赤松德赞在十三岁时执掌国政」
- 「太子木赤执政」

**10. 离藏 `departure`**
- 「莲师从贡塘拉土骑着狮子或骏马而去……到达了妙拂洲」
- 「堪布与译师又返回故里」——罗色措与特乐色未能传法而离去
- 「静命堪布说："那么我暂时去尼泊尔境内"」

**11. 弘法利生 `dharma_activity`**
- 「树立十善法、重新创立了以在家道德规范十六条为主善妙的二规法令」
- 「尼占帕翁卡地方也出现了一百位左右修行观音法、头挽发髻的在家瑜伽士」
- 「鲁墨大师培养出四顶梁之柱的弟子、八大栋梁弟子……由此使衰落的佛法从根本上得以恢复」

**12. 毁灭佛教 `dharma_destruction`**
- 「朗达玛秃头王以邪愿的牵引，邪魔入心，毁灭了佛教」
- 「拉隆·拜吉多吉降伏了恶王朗达玛」——止息灭法

**13. 恢复佛教 `dharma_restoration`**
- 「君王益西嘉村等在卫藏为了重新恢复衰退的佛法，委派来自卫藏的鲁墨·赤诚西绕等」
- 「创立了拉木、绕吉、也巴、瓦耶、凑迪等无数的道场」
- 「高僧玛尔·释迦牟尼、约·格威穹及藏·绕色三人把律藏驮在一头骡子上，前往葛逻和霍尔境」

**14. 婚姻联姻 `marriage`**
- 「迎娶了颦眉度母的化身——尼泊尔赤尊公主以及度母的化身——汉族文成公主」
- 「他和岩罗刹女交配，由此繁衍藏人」——起源传说中的结合

### 类型归属原则

一个事件只标注一个主类型。当多个类型可选时：
1. **降伏非人** 优先级最高（凡涉及降伏鬼神/非人的归入降伏非人）
2. **出家受戒** 次之（凡涉及戒律传授的归入出家受戒）
3. 其余按事件**核心行为**判断

## 三、事件记录格式

### YAML Schema

```yaml
event_id: "ch03-001"          # 格式: ch{章节}-{三位序号}
type: subjugation              # 事件类型代码（见第二节）
title: "莲师降伏十二地母"       # 4-8 字概括

participants:                  # 事件参与者（entity 引用）
  - entity_ref: "莲花生大士"   # 引用 entity_taxonomy 中的实体
    role: agent                # 角色: agent(施动)/patient(受动)/witness/beneficiary
  - entity_ref: "十二地母"
    role: patient

location: "藏地"               # 地点（entity 引用或自由文本）
time: "赤松德赞时期"            # 时间（原文表述，不做换算）

layers:                        # 叙述层标签（引用 narrative-layers）
  - layer: 神变
    author_stance: 认可         # 认可/中立/怀疑/反驳
    sub_layer: null            # 多异说时用自由文本区分

confidence: 0.90               # 置信度 [0.0, 1.0]
source_quote: "莲师降伏了十二地母、十三革绕神、二十一优婆塞为主的一切大力非人"
description: "莲师踏遍整个藏地，降伏十二地母等大力非人，使他们立誓守护正法。"
```

### 字段说明

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `event_id` | Y | string | `ch{NN}-{NNN}`，按出现顺序编号 |
| `type` | Y | enum | 14 个类型代码之一 |
| `title` | Y | string | 4-8 字概括 |
| `participants` | Y | list | 至少一个参与者 |
| `participants[].entity_ref` | Y | string | 实体名称，须在 entity 库中可查 |
| `participants[].role` | Y | enum | `agent` / `patient` / `witness` / `beneficiary` / `recipient` |
| `location` | N | string | 地点；无值时填 `null` |
| `time` | N | string | 保留原文表述；无值时填 `null` |
| `layers` | Y | list | 至少一个叙述层 |
| `layers[].layer` | Y | enum | 七层之一：史传/典据/授记/宗派/神变/证境/传闻 |
| `layers[].author_stance` | Y | enum | 认可/中立/怀疑/反驳；默认中立 |
| `layers[].sub_layer` | N | string | 多异说区分用；原则 #21 |
| `confidence` | Y | float | `[0.0, 1.0]`，标注者评估 |
| `source_quote` | Y | string | 原文引用 |
| `description` | N | string | 补充描述（2-4 句） |

### 多角色事件示例（原则 #20）

出家仪式涉及多角色，用一个 record + 角色字段：

```yaml
event_id: "ch03-012"
type: ordination
title: "预试七人出家"
participants:
  - entity_ref: "巴赤则"
    role: recipient              # 受戒者
  - entity_ref: "静命"
    role: agent                  # 亲教师
    ritual_role: "亲教师"
  - entity_ref: "达那西拉"
    role: agent
    ritual_role: "羯磨师"
  - entity_ref: "则那莫札"
    role: agent
    ritual_role: "屏教师"
location: "桑耶寺"
time: "赤松德赞时期"
layers:
  - layer: 史传
    author_stance: 认可
confidence: 0.95
source_quote: "菩提萨埵作为亲教师，达那西拉和则那莫札分别作为羯磨师与屏教师……巴赤则出家为僧并受了比丘戒"
```

### 物体作为事件主体示例（原则 #19）

```yaml
event_id: "ch03-018"
type: dharma_activity
title: "佛像开口说话"
participants:
  - entity_ref: "札马珍桑殿堂佛像"  # 法器/圣物类 entity
    role: agent
layers:
  - layer: 神变
    author_stance: 认可
confidence: 0.85
source_quote: "第二天早晨寺庙里所有佛像享用神馐并且开口说话"
```

## 四、与 narrative-layers 的协同

每个事件 record 必须带 `layers` 字段，引用 `narrative-layers.md` v0.2 的七层标签。

### 事件类型与典型叙述层映射

| 事件类型 | 典型层 | 说明 |
|----------|--------|------|
| 即位执政 | 史传 | 凡常政治行为 |
| 出家受戒 | 史传 | 凡常宗教行为 |
| 建寺 | 史传 / 神变 | 建造本身史传；"晚上非人建得更高"则神变 |
| 迎请 | 史传 | 凡常行为；若佛像自行飞来则神变 |
| 灌顶传法 | 史传 | 传法本身凡常；密法内容不影响事件层 |
| 译经 | 史传 | 凡常文化行为 |
| 降伏非人 | 神变 | 必须诉诸密法本体论 |
| 授记 | 授记 (+ 典据) | 授记既是层也是事件类型（原则 #18） |
| 埋藏伏藏 | 神变 (+ 授记) | 伏藏埋藏诉诸密法本体论 |
| 离藏 | 史传 / 神变 | 步行离去=史传；腾空而飞=神变 |
| 弘法利生 | 史传 | 凡常弘法活动 |
| 毁灭佛教 | 史传 | 凡常政治/宗教事件 |
| 恢复佛教 | 史传 | 凡常宗教活动 |
| 婚姻联姻 | 史传 | 凡常家族行为 |

### 关键规则

1. **层标签标注事件本身，不是事件主体**——莲师建桑耶寺（凡常行为部分）= 史传，莲师降伏鬼神 = 神变。主体是谁不决定层
2. **授记双重身份**（原则 #18）——"松赞干布授记赤松德赞"既是 `prophecy` 类型事件，也带 `授记` 叙述层标签，两者不冲突
3. **一个事件可多层**——"莲师隐藏伏藏并授记开取时间"同时带 `神变` + `授记` 两个层标签
4. **史传与神变互斥**——同一事件的同一断言不能同时打这两个层（`narrative-layers.md` 原则 #3）

## 五、与 entity-taxonomy 的协同

事件的 `participants` 字段通过 `entity_ref` 引用 `entity-taxonomy.md` v0.4 中定义的实体。

### 实体引用规则

1. **entity_ref 必须对应已抽取的实体**——来自 `kg/entities/` 或 `kg/poc/` 的 ~191 个 entity
2. **角色不是类目**（entity-taxonomy 原则 #7）——亲教师/羯磨师/译师等角色在事件 record 中用 `role` 或 `ritual_role` 表达，不影响实体分类
3. **通称非人不作 entity**（原则 #16）——"凶神恶煞""鬼神"等无名非人不在 `participants` 中建 entity_ref，改在 `description` 文本中描述
4. **具名非人需 entity_ref**——十二地母、无恼龙等有名称的非人需要引用对应的 entity
5. **物体可作为参与者**（原则 #19）——法器/圣物类 entity 可出现在 `participants` 中，`role` 为 `agent`
6. **集合 entity 可作为参与者**——预试七人、师君三尊等集合 entity 可整体出现在 `participants` 中

### 事件类型与常见实体类目对应

| 事件类型 | 常见 agent 类目 | 常见 patient/recipient 类目 |
|----------|----------------|---------------------------|
| 迎请 | 人物.帝王 / 人物.大臣 | 人物.法师 / 法器.佛像 |
| 出家受戒 | 人物.法师（传戒者） | 人物.法师（受戒者） |
| 建寺 | 人物.帝王 / 人物.法师 | 寺院 |
| 灌顶传法 | 人物.法师 | 人物.帝王 / 人物.法师 |
| 译经 | 人物.法师（译师） | 经典 |
| 降伏非人 | 人物.法师 | 非人 |
| 授记 | 圣众 / 人物.法师 | 人物 / 教法 |
| 埋藏伏藏 | 人物.法师 | 经典.密续.岩藏 |
| 即位执政 | 人物.帝王 | — |
| 离藏 | 人物.法师 | — |
| 毁灭佛教 | 人物.帝王 | 寺院 / 教法 |
| 恢复佛教 | 人物.法师 / 人物.帝王 | 教法 / 教派 |
| 婚姻联姻 | 人物.帝王 | 人物.王妃 |

## 六、已知局限

| # | 局限 | 说明 |
|---|------|------|
| 1 | **时间粒度粗糙** | 第三品多为相对时间（"X 时期""此后第五代"），绝对年份极少。`time` 字段保留原文表述，不做推算 |
| 2 | **事件类型仅覆盖第三品** | 14 个类型从第三品归纳，后续章节（远传经幻心、近传伏藏史等）可能需要新增类型（如"掘取伏藏""转世认定""净相得法"等） |
| 3 | **通称非人信息损失** | 原则 #16 下无名非人不建 entity，降伏事件中一部分参与者只能出现在 description 文本中 |
| 4 | **多异说事件尚无标准化** | 莲师驻藏时长（短/中/长时间观点）等多异说场景，暂用 `sub_layer` 自由文本区分，未来可能需更结构化的表达 |
| 5 | **因果链未建模** | 当前 schema 不包含事件间的因果关系字段。如"奸臣劝请 → 莲师事业中止"等因果链需要后续扩展 |
| 6 | **事件粒度需实际标注验证** | 本草案中的粒度标准为理论设计，需在实际标注第三品全部事件后校准 |
